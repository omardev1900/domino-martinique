1. Interdire le multijoueur aux invités (Le plus facile)
Pourquoi : C'est un simple "garde-fou" au niveau de l'interface utilisateur.

Action : Dans ton LobbyScreen, avant de lancer la fonction handleJoinRoom ou handleCreateRoom, tu vérifies simplement auth.currentUser.isAnonymous.

Résultat immédiat : Cela protège tes serveurs et incite à la création de compte.

2. Système de Coins (Facile à Moyen)
Pourquoi : C'est une base de données simple. Tu as déjà une collection users, il suffit d'y ajouter un champ coins: number.

Action : Créer deux fonctions utilitaires : addCoins(amount) et subtractCoins(amount).

Complexité : La seule difficulté est de s'assurer que l'ajout de coins est bien sécurisé côté Firebase (règles de sécurité) pour éviter que quelqu'un ne se donne des millions de coins via la console.

3. Suppression de compte avec anonymisation (Moyen)
Pourquoi : On l'a déjà bien balisé ensemble. C'est un flux de données (Auth + Firestore).

Action : Implémenter le "Soft Delete" (anonymisation) que nous avons décrit : supprimer l'email et le nom, mais garder l'UID et le flag isDeleted: true.

Difficulté : Gérer l'OTP par email demande une petite configuration de service d'envoi d'emails.

4. Bugs de Victoire/Score non comptabilisés (Difficile)
Pourquoi : C'est le cœur du moteur de jeu (LogicEngine). Si une victoire n'est pas comptée, c'est que la condition de fin de manche ne se déclenche pas ou que l'état du jeu (State) est perdu pendant la transition.

Action : Il faut traquer l'endroit où la fonction checkWinner est appelée et vérifier pourquoi elle retourne null alors qu'un joueur a gagné.

5. Mode Cochon et calcul des points (Le plus difficile)
Pourquoi : Les règles du domino martiniquais (Cochon, points doublés, etc.) sont algorithmiquement complexes. Un bug ici signifie qu'il faut revoir les formules mathématiques dans ton ScoringEngine.

Action : Créer des tests unitaires très précis pour chaque cas de figure (Cochon simple, Cochon sur boudé, etc.) pour voir où le calcul dévie.

Ma recommandation de feuille de route :
Matin : Bloquer les invités + Ajouter le champ "Coins" (tu auras un sentiment d'accomplissement rapide).

Après-midi : Plonger dans les Bugs de Score et de Victoire. C'est le plus critique pour tes joueurs. Si le score est faux, les joueurs ne resteront pas, même s'ils ont des coins.

Fin de journée / Demain : La suppression de compte (c'est une fonction de gestion, moins urgente que le gameplay).
